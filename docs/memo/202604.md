> ハーネスエンジニアリングは、AI コーディングエージェントを“自由に走らせる”のではなく、守るべき制約を明示し、逸脱したら検知・修正・停止できるようにする考え方です。名前のとおり、「手綱（harness）」のメタファーに近いものです。

## Vim と Biome とか oxfmt とか

Vite Plus(oxfmt) やら Biome やらで Prettier ライクなフォーマッタが色々出てきていて、作業中のプロジェクトがどのフォーマッタを利用しているかが様々。

一方で自分の Vim だと fork した vim-prettier を利用し続けているが、こいつは Biome や oxfmt に対応していないのでそろそろ辛い。
vim-prettier のコードを軽く読んだ(Claude に読ませた) ところ、 `:system` で Prettier の CLI に標準入力でバッファの内容渡している程度なので、拡張すれば他フォーマッタのバイナリ使わせるのもそう難しくはなさそう。

`./node_modules/.bin` に複数のフォーマッタが見つかった場合の優先度をどうするかがちょっと悩みどころか。

- Prettier より Biome, oxfmt が優先
- oxfmt より Vite Plus が優先
- Vite Plus より Biome が優先

こんな感じか？
