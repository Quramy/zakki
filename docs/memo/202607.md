## Karabiner-Elements の設定ミス

今この雑記帳を書いていて気づいたが、先日に mac os tahoe にアップグレートして以降、Karabiner-Elements が意図通りに動作していなかったことに気づいた。
ESC を押したら入力ソースをデフォルトの半角英数字用のものに切り替えるようにしていた。
Vim でインサートモードで日本語入力している最中にノーマルモードに ESC 一発で戻りたい、という意図。

今使っている端末をキッティングしたときに、適当に ESC to en US という設定をどこからか持ってきて有効にしていたのだが、おそらく入力ソース名が変わったか何かで動かなくなっていたのかな。。

https://ke-complex-modifications.pqrs.org/#japanese からインストールできる「esc キーを押したときに、英数キーも送信する（vim 用）」というものを使うように変更。
こちらは、英数キーを追加発動させる、という設定であるため、入力ソース名に依存視せずに動作する。

```json
{
  "description": "escキーを押したときに、英数キーも送信する（vim用）",
  "manipulators": [
    {
      "from": { "key_code": "escape" },
      "to": [{ "key_code": "escape" }, { "key_code": "japanese_eisuu" }],
      "type": "basic"
    }
  ]
}
```

## Phantom Squatting について

https://thehackernews.com/2026/07/phantom-squatting-uses-ai-hallucinated.html にて、Phantom Squatting という term を知る。

LLM が生成しそうなドメインを狙ってフィッシング攻撃の対象とする、というニュアンスという理解。
すでに複数のモデルで悪意のあるサイトとして運用されてしまっているドメインを生成している、とのこと。

> To measure the problem, Unit 42 asked two AI models 685,339 questions about 913 well-known brands across technology, finance, healthcare, government, gambling, and other sectors.
>
> The models produced 2.1 million links. Threat intelligence already flagged 13,229 of them as outright malicious, meaning the AI was handing out known-bad addresses

その割合は 13,229/2,100,000 なので、0.5 % 程度ということになる。
