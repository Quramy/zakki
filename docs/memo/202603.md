# 3/17

## `getComputedAccessibleNode` が死んでいた件

数年前の知識で止まっていたが、ブラウザから AOM の情報を抜くための API (e.g. `getComputedAccessibleNode`) を公開する計画は頓挫していた模様。

https://github.com/WICG/aom/blob/gh-pages/explainer.md#what-happened-to-accessiblenode

そもそもが WICG の目的としては、読み取りだけではなく、JavaScript 側から AOM の構築もできるようにする計画であったらしい。
おそらく、描画系に Canvas を使っておき、仮想的な AOM Node を設定させることでパフォーマンスとセマンティクスが両立したアプリケーションが作れる、といったような目論見だったのだろう。

API の仕様化に取り組む過程で、以下などの課題が生じた模様。

- 書き込みロジック
- ブラウザ間の差異をどのように吸収するか

読み取り専用 API として、https://github.com/WICG/aom/issues/197 が Open なままではあるが、もう 3 年近く動きがないし、ブラウザ間の結果差異を許容すると Fingerprinting リスクが突っこまれてセキュリティ方面から懸念も出そう。
テストの目的であれば、CDP が対応していれば多分十分、ということになりそう。

## React Compiler の副作用

https://kakehashi-dev.hatenablog.com/entry/2026/03/24/090000

上記に、React Compiler 適用時の注意事項について書いてあった。
RHF などの一部ライブラリが以下の前提とそぐわないために意図しない副作用が生じるとのこと。

> React Compilerはpropsやstateの参照が同じなら、出力も同じという前提で自動メモ化を行います。つまり、オブジェクトの参照が変わっていなければ再レンダリングをスキップします。

他には Tanstack Query でも同様の問題が発生するらしい。
